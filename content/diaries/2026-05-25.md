## Q？

Streamed responses 和 Unstreamed responses 各优势劣势在哪？

## A！

Streamed and Unstreamed responses describe how an AI agent sends its answer to the user.

### Streamed responses
- The agent stars sending words as soon as it generates them. (当 agent 一开始生成单词，就会立即发送)
- The user sees the text grow on the screen in real time. (用户能能在屏幕实时看到文本的增长)
- The feel fast and lets the user stop or change the request early. (感觉上很快，让用户能提前停止或改变请求)
- It is useful for long answers and chat-like-apps. (更适合长回答和聊天类软件)

### Unstreamed responses
- An unstreamed responses waits until the whole answer is ready, then sends it all at once. (非流失响应等待所有回答准备好后，一次性发送所有内容)
- This makes the code on the client side simpler and is easier to cache or log, but the user must wait longer, especially for big output. (可以让客户端代码更简单，缓存和log更容易，但是用户必须等待更长的时间，尤其是大量输出)

Choosing between the two depends on the need for speed, the length of the answer, and how complex you want the client and server to be.
